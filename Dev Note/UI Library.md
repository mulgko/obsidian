# UI Library

라이브러리 성격이라는 것은 특정 비즈니스 로직(데이터)에 종속되지 않고, 어느 프로젝트를 가져다놓아도 바로 쓸 수 있는 범용적인 UI 부품을 의미 

1. 라이브러리 성격의 특징
	- 데이터를 모름: "게시글 목록", "사용자 프로필" 같은 구체적인 내용을 담지 않는다. 대신 text, children, color와 같은 기초적인 값만 props로 받습니다. 
	- 재사용성이 극대화: 버튼 하나를 만들면 로그인 페이지, 글쓰기 페이지, 마이페이지 등 어디서든 똑같이 쓸 수 있습니다.
	- 디자인 시스템의 기초: 서비스의 전체적인 룩앤필(색상, 여백, 폰트 등)을 결정하는 가장 작은 단위입니다.

2. 구체적인 예시
	- Button:  클릭 기능과 스타일만 가짐 (로그인 버튼인지, 삭제 버튼인지는 부모가 결정)
	- Input: 글자를 입력받는 폼만 제공
	- Badge: 태그나 상태를 표시하는 작은 칩
	- Modal/Dialog: 내용을 담을 빈 창 (안에 뭐가 들어갈지는 매번 다름)

-----------------------------------------------------------------

반대로 라이브러리 성격이 아닌 것은?
- Header(components/common): 디자인 요소만 있는 게 아니라, 현재 로그인한 사용자 정보나 서비스 로고 등 우리 서비스의 공통 로직이 들어가 있습니다. 
- PostCard: 이건 UI 부품처럼 보이지만, 내부에 post.title, post.author처럼 앱의 데이터 구조를 알아야지만 쓸 수 있습니다.
